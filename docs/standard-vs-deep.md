# Standard vs Deep

Use this page to explain packaging without marketing fog.

## Compare

- Scope of analysis
- Turnaround expectations
- Evidence depth
- Decision confidence implications
- Ideal use cases

## Keep category separate

Standard and Deep describe execution profile and evidence depth. They do not replace the category-routing policy.

Before deciding that Deep is needed, also confirm which scientific category the staged package actually supports in [Category Policy and Routing](./category-policy-and-routing.md).

## Decision rule

A user should know when Standard is sufficient and when Deep is justified.
